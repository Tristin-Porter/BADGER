# x86_32 Architecture

## Overview

x86_32 (also known as x86, IA-32, i386) is the 32-bit x86 architecture. It shares most characteristics with x86_64 but uses 32-bit registers and addresses.

## Register Set

### General Purpose Registers (32-bit)
- `eax`, `ebx`, `ecx`, `edx` - Accumulator, base, count, data
- `esi`, `edi` - Source and destination index
- `ebp`, `esp` - Base pointer and stack pointer

### Stack Simulation
- `ebx` - Stack slot 0
- `ecx` - Stack slot 1
- `edx` - Stack slot 2
- `edi` - Stack slot 3
- `[ebp - N]` - Spilled values

## Key Differences from x86_64

1. **No REX prefix** - Uses standard x86 encoding
2. **32-bit registers** - eax instead of rax
3. **32-bit addresses** - 4GB address space
4. **No r8-r15** - Only 8 general-purpose registers

## Calling Convention

```asm
push ebp
mov ebp, esp
; Function body
mov esp, ebp
pop ebp
ret
```

## Common Instructions

Same mnemonic as x86_64 but with 32-bit registers:
- `mov eax, ebx`
- `add eax, 42`
- `push ebp`
- `pop ebp`
- `ret`

## Instruction Encoding

Similar to x86_64 but:
- No REX prefix
- Default operand size is 32-bit
- Addressing uses 32-bit values
