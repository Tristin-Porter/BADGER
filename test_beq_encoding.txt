B.cond encoding in ARM64:
Binary: 0101010 0 [imm19] 0 [cond]
        bits 31-25 24 23-5  4  3-0

For B.EQ (cond=0000) with offset -1:
imm19 = -1 = 0x7FFFF (19-bit two's complement)
cond = 0000

Full instruction (32 bits, big-endian representation):
0101010 0 1111111111111111111 0 0000
= 0101 0100 1111 1111 1111 1111 1110 0000
= 0x54FFFFE0

In little-endian bytes:
Byte 0: E0 (lowest 8 bits: ...11110000 - contains cond in bits 3-0)
Byte 1: FF
Byte 2: FF
Byte 3: 54 (highest 8 bits)

So in code array for 2 instructions:
Index 0-3: NOP instruction
Index 4-7: B.EQ instruction
  Index 4: E0 (contains cond in bits 3-0)
  Index 5: FF
  Index 6: FF
  Index 7: 54

Condition code is in beqCode[4] & 0x0F!
The test is correct!
